# trust-before-touch-so101

Simulation-first robotics security MVP for a 3-arm SO-101 setup using a LeRobot-oriented integration layer.

## Overview
This project implements a physical challenge-response system (PCS) where a prover arm must execute randomized motions generated by a leader arm while a verifier arm + camera observes execution. The system computes trust and decides pass/fail before handoff-ready state.

## Architecture
- **FastAPI backend**: protocol orchestration, REST + WebSocket streaming.
- **State machine**: `idle -> claim_received -> challenge_issued -> executing -> verifying -> passed/failed/aborted`.
- **Simulation engine**: deterministic challenge execution with normal/replay/delay modes.
- **Scoring**: weighted trust breakdown (trajectory/timing/observation/alignment).
- **Persistence**: SQLite session + event log.
- **Dashboard**: simple HTML/JS UI at `/`.
- **Hardware layer**: LeRobot-oriented interfaces with simulation adapters and SO-101 stubs.

## LeRobot integration notes
Simulation uses interfaces in `hardware/interfaces.py` and `hardware/simulated.py`. Real SO-101 support is prepared via `hardware/lerobot_so101_stub.py`, where LeRobot control + camera hooks should be implemented.

## Limitations
- No real robot control in MVP.
- Vision/contact are mocked.
- Attack detection is heuristic.
